### Тема 1. Введение в базы данных и модели данных

1.  **Что такое База Данных (БД)?** — Совокупность взаимосвязанных данных, организованных по схеме.
2.  **Что такое СУБД?** — Программное обеспечение для создания, управления и манипулирования базами данных.
3.  **В чем главное отличие БД от файловой системы?** — Наличие схемы данных, поддержка целостности и структурированный доступ.
4.  **Что такое «Отношение» (Relation) в реляционной модели?** — Таблица.
5.  **Как называется строка в реляционной таблице?** — Кортеж (Запись).
6.  **Как называется столбец в реляционной таблице?** — Атрибут (Поле).
7.  **Что такое Домен атрибута?** — Множество допустимых значений для столбца.
8.  **Что такое Схема отношения?** — Набор имен атрибутов (заголовок таблицы).
9.  **Что такое Кардинальность отношения?** — Количество строк (кортежей) в таблице.
10. **Что такое Степень (арность) отношения?** — Количество столбцов (атрибутов) в таблице.
11. **Какой тип модели данных представляет собой дерево?** — Иерархическая.
12. **Какой тип модели данных представляет собой граф (произвольные связи)?** — Сетевая.
13. **Пример СУБД типа «Ключ-Значение»?** — Redis.
14. **Пример документо-ориентированной СУБД?** — MongoDB.
15. **Какой формат данных популярен в NoSQL?** — JSON.
16. **Что означает физическая независимость данных?** — Изменение способа хранения на диске не влияет на логическую структуру и программы.
17. **Кто считается автором реляционной модели?** — Эдгар Ф. Кодд.
18. **Что такое OLTP?** — Системы обработки транзакций в реальном времени (быстрые, частые операции).
19. **Что такое OLAP?** — Системы аналитической обработки данных (сложные отчеты).
20. **Могут ли в реляционном отношении быть одинаковые кортежи?** — Нет (в строгой теории), в SQL — да (если не использовать DISTINCT/Unique).

### Тема 2. Проектирование баз данных

21. **Какова цель инфологического проектирования?** — Создание модели предметной области (например, ER-диаграммы), понятной человеку.
22. **Что такое Сущность (Entity)?** — Объект реального мира, о котором хранятся данные.
23. **Что такое Первичный ключ (Primary Key)?** — Минимальный набор атрибутов, уникально идентифицирующий строку.
24. **Может ли Первичный ключ содержать NULL?** — Нет.
25. **Что такое Внешний ключ (Foreign Key)?** — Атрибут, ссылающийся на первичный ключ другой таблицы для связи.
26. **Что такое Суррогатный ключ?** — Искусственный идентификатор (обычно автоинкремент), не имеющий смысла в реальном мире.
27. **Тип связи, когда одной записи А соответствует много записей Б?** — «Один-ко-многим» (1:M).
28. **Тип связи, когда записи А соответствует одна запись Б и наоборот?** — «Один-к-одному» (1:1).
29. **Как реализуется связь «Многие-ко-многим» (M:M) в реляционной БД?** — Через промежуточную (связующую) таблицу.
30. **Что такое ER-диаграмма?** — Графическое представление сущностей и связей ("Сущность-Связь").
31. **Что такое Нормализация?** — Процесс организации данных для устранения избыточности и аномалий.
32. **Какое требование у 1-й нормальной формы (1НФ)?** — Атомарность значений (в ячейке одно значение, нет списков).
33. **Какое требование у 2-й нормальной формы (2НФ)?** — Таблица в 1НФ + все неключевые атрибуты зависят от *всего* первичного ключа.
34. **Какое требование у 3-й нормальной формы (3НФ)?** — Таблица во 2НФ + нет транзитивных зависимостей (неключевые атрибуты не зависят друг от друга).
35. **Что такое Денормализация?** — Намеренное нарушение нормальных форм для ускорения чтения данных.
36. **Что такое Целостность сущностей?** — У каждой строки должен быть уникальный идентификатор (PK).
37. **Что такое Ссылочная целостность?** — Значение внешнего ключа должно ссылаться на существующий первичный ключ или быть NULL.
38. **Что делает стратегия `ON DELETE CASCADE`?** — При удалении родительской записи автоматически удаляет связанные дочерние.
39. **Что делает стратегия `ON DELETE RESTRICT`?** — Запрещает удаление родительской записи, если есть связанные дочерние.
40. **Что такое NULL с точки зрения БД?** — Отсутствующее или неизвестное значение (не равно нулю или пустой строке).
41. **Какая кардинальность означает обязательное участие в связи?** — 1 (или 1..N).
42. **Какая кардинальность означает необязательное участие в связи?** — 0 (или 0..N).
43. **Как обозначается «многие» в нотации Чена?** — Буквами M или N.
44. **Что такое Атрибут-характеристика?** — Атрибут, описывающий свойство сущности (например, Цвет, Вес).
45. **Зачем нужны CASE-средства при проектировании?** — Для автоматизации построения моделей (диаграмм) и генерации SQL-кода.

### Тема 3. Язык SQL

46. **К какой группе относится команда `CREATE TABLE`?** — DDL (Data Definition Language).
47. **К какой группе относится команда `SELECT`?** — DML (Data Manipulation Language).
48. **К какой группе относится команда `GRANT`?** — DCL (Data Control Language).
49. **Команда для получения данных из таблицы?** — `SELECT`.
50. **Команда для добавления новых строк?** — `INSERT INTO`.
51. **Команда для изменения существующих данных?** — `UPDATE`.
52. **Команда для удаления строк?** — `DELETE FROM`.
53. **Команда для полного удаления таблицы (со структурой)?** — `DROP TABLE`.
54. **Команда для быстрой очистки таблицы (данных) без удаления структуры?** — `TRUNCATE TABLE`.
55. **Как выбрать уникальные значения столбца?** — `SELECT DISTINCT`.
56. **Как фильтровать записи по условию?** — Клауза `WHERE`.
57. **Как сортировать записи?** — Клауза `ORDER BY`.
58. **Порядок сортировки по убыванию?** — `DESC`.
59. **Какой оператор ищет по текстовому шаблону?** — `LIKE`.
60. **Что означает знак `%` в операторе `LIKE`?** — Любое количество любых символов.
61. **Что означает знак `_` в операторе `LIKE`?** — Ровно один любой символ.
62. **Как проверить значение на NULL?** — `IS NULL` (нельзя использовать `= NULL`).
63. **Как проверить вхождение значения в список?** — Оператор `IN (...)`.
64. **Как проверить вхождение в диапазон?** — Оператор `BETWEEN`.
65. **Агрегатная функция для подсчета строк?** — `COUNT()`.
66. **Агрегатная функция для суммы?** — `SUM()`.
67. **Как сгруппировать данные для агрегации?** — Клауза `GROUP BY`.
68. **Как фильтровать данные ПОСЛЕ группировки (по агрегатам)?** — Клауза `HAVING`.
69. **Какой JOIN возвращает только совпадающие строки?** — `INNER JOIN`.
70. **Какой JOIN возвращает все строки левой таблицы + совпадения из правой?** — `LEFT JOIN`.
71. **Какой JOIN возвращает декартово произведение (все со всеми)?** — `CROSS JOIN`.
72. **Оператор объединения результатов запросов с удалением дубликатов?** — `UNION`.
73. **Оператор объединения результатов с сохранением дубликатов?** — `UNION ALL`.
74. **Что такое Подзапрос?** — Запрос, вложенный в другой запрос (например, в WHERE или FROM).
75. **Для чего используется `AS`?** — Для создания псевдонимов (алиасов) таблиц или столбцов.

### Тема 5. Транзакции и управление

76. **Что такое Транзакция?** — Логически неделимая последовательность операций с БД.
77. **Что означает буква A в ACID?** — Atomicity (Атомарность) — все или ничего.
78. **Что означает буква C в ACID?** — Consistency (Согласованность) — данные корректны до и после.
79. **Что означает буква I в ACID?** — Isolation (Изолированность) — транзакции не мешают друг другу.
80. **Что означает буква D в ACID?** — Durability (Долговечность) — данные сохранены после фиксации.
81. **Команда для подтверждения транзакции?** — `COMMIT`.
82. **Команда для отмены транзакции?** — `ROLLBACK`.
83. **Что такое SAVEPOINT?** — Точка сохранения внутри транзакции для частичного отката.
84. **Что такое «Грязное чтение» (Dirty Read)?** — Чтение данных, добавленных другой незавершенной транзакцией.
85. **Что такое «Неповторяющееся чтение»?** — Когда повторное чтение той же строки в транзакции дает другой результат (кто-то изменил данные).
86. **Что такое «Фантомное чтение»?** — Когда повторный запрос диапазона возвращает разное количество строк (кто-то добавил/удалил данные).
87. **Самый низкий уровень изоляции?** — Read Uncommitted.
88. **Уровень изоляции по умолчанию во многих СУБД?** — Read Committed.
89. **Уровень изоляции, предотвращающий неповторяющееся чтение?** — Repeatable Read.
90. **Самый строгий уровень изоляции (нет фантомов)?** — Serializable.
91. **Что такое Взаимная блокировка (Deadlock)?** — Ситуация, когда две транзакции ждут ресурсы друг друга бесконечно.
92. **Что такое Пессимистичная блокировка?** — Блокирование данных перед началом работы с ними.
93. **Что такое Оптимистичная блокировка?** — Проверка версий данных перед записью (без блокировки при чтении).
94. **Команда для создания пользователя?** — `CREATE USER`.
95. **Команда для выдачи прав?** — `GRANT`.
96. **Команда для отзыва прав?** — `REVOKE`.
97. **Что такое Роль (Role) в БД?** — Именованная группа привилегий для удобства администрирования.
98. **Принцип наименьших привилегий — это...** — Выдавать пользователю только те права, которые необходимы для работы.
99. **Что такое SQL-инъекция?** — Атака, при которой вредоносный код внедряется в запрос к базе.
100. **Что такое Представление (View) в контексте безопасности?** — Способ скрыть реальную структуру таблиц и ограничить доступ к определенным столбцам/строкам.
